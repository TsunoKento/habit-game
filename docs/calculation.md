# 計算ロジック

## 累計経験値

各習慣の累計経験値は以下で求める。経験値はDBに保存せず、達成記録から毎回動的に算出する。

- 基本経験値の合計（= 達成回数 × exp_per_done）
- 連続達成ボーナスの合計

### ボーナスの再計算方法

1. 達成記録を日付昇順で走査し、連続達成ストリークを追跡する
2. 前日との日付差が1日を超えた場合、連続が途切れたとみなしストリークを1にリセットする
3. ストリーク日数がボーナス閾値（3, 7, 14, 30）と一致した日にボーナスEXPを加算する
4. ストリークがリセットされ、再び閾値に到達した場合は再度ボーナスを付与する

---

## 総合経験値

全習慣の累計経験値の合計値とする。

---

## 現在レベル

レベルテーブルをもとに、現在の累計経験値が到達している最大レベルを返す。

### レベルテーブル

| レベル | 必要累計EXP |
|---|---:|
| 1 | 0 |
| 2 | 30 |
| 3 | 70 |
| 4 | 120 |
| 5 | 200 |
| 6 | 300 |

---

## 連続達成ボーナス

連続達成日数に応じて、追加経験値を付与する。

### ボーナステーブル

| 連続達成日数 | ボーナスEXP |
|---|---:|
| 3日 | 5 |
| 7日 | 10 |
| 14日 | 20 |
| 30日 | 50 |

### ルール

- ボーナスは該当日数に到達した日にのみ付与する（例: 3日連続達成した日に +5EXP）
- 4日目以降は、別の閾値に達するまで追加ボーナスは発生しない
- ボーナスは閾値に到達するたびに繰り返し付与される
  - 例: 3日目 +5EXP、途切れてリセット後、再び3日目に +5EXP
  - 例: 連続6日達成した場合、3日目に +5EXP、6日目は閾値に該当しないためボーナスなし

---

## 連続達成日数

対象習慣について、過去に向かって日付が連続してレコードが存在する日数を数える。

1. 起点は「当日のレコードがあれば当日、なければ前日」とする
2. 起点から1日ずつ遡り、レコードが存在する限りカウントを増やす
3. レコードが存在しない日（空白日）が現れた時点でカウントを停止する
4. 起点にレコードが存在しない場合は 0 を返す
